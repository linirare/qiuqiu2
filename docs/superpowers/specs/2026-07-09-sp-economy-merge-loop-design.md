# SP 经济驱动的合并循环 · 设计文档

> 日期：2026-07-09
> 目标：将 Build 环节从"零代价纯收益"改为 SP 驱动的多向权衡，让每一刻都有策略张力

---

## 问题诊断

当前 Build 循环的两个结构性问题：

1. **合并没有策略性**：有对子就合，零代价纯收益，没有"该不该合"的权衡
2. **合并战力感知弱**：高级球和低级球产出的兵差距不够大，合了半天感觉不到变强

根因：合并无成本 → 无抉择 → 无感觉。需要引入稀缺资源制造张力。

## 参考模型：球球英雄

- SP 是万物燃料：召唤、升级、合球都要 SP
- 合并减少场上球数，火力覆盖变弱，需要权衡
- 棋盘满时必须合才能继续
- 核心张力："SP 永远不够用"

## 设计方案

### 改动一：取消自动补球，改为手动 SP 召唤

**现状**：每 4.4 秒自动在空位补一个随机球。

**改为**：
- 自动补球逻辑完全移除（`main.js` 中 `ballTimer` + `autoSpawnBall`）
- 玩家**点击空格子** → 消耗 1 SP → 从卡组中随机抽一个水果放入该格
- 如果 SP 为 0，点击空格无反应（或飘字"果汁不足"）

**涉及的代码**：
- `main.js` update()：移除 `ballTimer` 和自动补球块
- `input.js` onDown()：空格点击检测，判断 SP > 0 时调用 summon
- `config.js`：可保留 `BALL_SPAWN_INTERVAL` 常量供参考

### 改动二：合并消耗 SP

**现状**：`tryMerge()` 零代价。两球拖到一起即合成。

**改为**：
- `tryMerge()` 执行前检查 SP >= 1
- SP 不足时返回 `{ merged: false, reason: 'no_sp' }`
- `input.js` onUp() 中合并分支接收此结果，SP 不足时飘字「果汁不足 · 无法合成」且不执行合成

**涉及的代码**：
- `board.js` tryMerge()：新增 SP 检查和消耗
- `input.js` onUp()：处理 SP 不足的反馈

### 改动三：SP 经济微调

| 参数 | 现值 | 改后 | 原因 |
|------|------|------|------|
| SP 被动回复间隔 | 3.6s | 3.0s | 新消耗点多，略微加速回复 |
| SP 回复上限 | 6 + spLv/2 | 8 + spLv/2 | 给玩家更多缓冲空间 |
| SP 上限 | 18 + spLv | 不变 | - |
| 击杀奖励 | +1 SP | 不变 | 已有正反馈循环 |

**涉及的代码**：
- `config.js`：调整 `SP_PASSIVE` 常量
- `state.js`：调整 `getSpRecoverCap()`

### 改动四（配套）：高级球战力差距拉大

合并有了代价之后，需要让合并结果**物有所值**。当前等级倍率 `LEVEL_MUL` 在 Lv3→Lv5 的增幅不够感知。

| 等级 | 当前倍率 | 改后倍率 |
|------|---------|---------|
| Lv1 | 1.0 | 1.0 |
| Lv2 | 1.58 | 1.65 |
| Lv3 | 2.34 | 2.60 |
| Lv4 | 3.32 | 4.00 |
| Lv5 | 4.62 | 6.00 |
| Lv6 | 6.30 | 8.50 |
| Lv7 | 8.50 | 12.00 |

Lv3→Lv5 从约 2 倍变为约 2.3 倍，合出来的高级球能明显改变一路的局势。

**涉及的代码**：
- `config.js`：调整 `LEVEL_MUL` 数组

### 改动五（配套）：出兵冷却随等级变化更明显

当前出兵间隔 `SPAWN_COOLDOWNS` 按等级递减但幅度不大。高级球产兵应该**明显更快**，让玩家感到"合上去就值了"。

| 等级 | 当前冷却 | 改后冷却 |
|------|---------|---------|
| Lv1 | 5.6s | 6.5s |
| Lv2 | 5.0s | 5.5s |
| Lv3 | 4.4s | 4.5s |
| Lv4 | 3.9s | 3.5s |
| Lv5 | 3.4s | 2.8s |
| Lv6 | 2.9s | 2.2s |
| Lv7 | 2.5s | 1.8s |

**涉及的代码**：
- `config.js`：调整 `SPAWN_COOLDOWNS` 数组

## 改动范围汇总

| 文件 | 改动类型 | 改动量 |
|------|---------|--------|
| `config.js` | 参数调整 | ~10 行 |
| `state.js` | getSpRecoverCap() 调整 | ~2 行 |
| `main.js` | 移除自动补球，新增空格召唤处理 | ~20 行 |
| `input.js` | 空格点击召唤 + 合并 SP 检查 | ~25 行 |
| `board.js` | tryMerge 增加 SP 参数 | ~8 行 |

**不改的**：渲染层、战斗系统、AI、音效、棋盘布局、UI 面板

---

## 实施顺序

```
Phase 1: 参数调整（config.js + state.js）—— 无依赖
Phase 2: 合并 SP 消耗（board.js + input.js）—— 无依赖
Phase 3: 手动召唤（main.js + input.js）—— 依赖 Phase 1 的 SP 参数
Phase 4: 测试验证
```

---

## 验收标准

1. 空格子不再自动出现新球
2. 点击空格子消耗 1 SP 并出现新球（从卡组随机）
3. SP 为 0 时点击空格无反应
4. 拖拽合并消耗 1 SP，SP 不足时合并不执行且飘字提示
5. SP 被动回复加快到 3.0s
6. Lv5 球的兵明显比 Lv3 强（约 2.3 倍），且出得更快
7. 整体节奏不会卡死——玩家总有事可做，不会长时间等 SP
